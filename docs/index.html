Installation

Uploading themes

Access your blog's settings

Select Design from the navigation menu on the left of your admin area.

Upload your theme

Next click on the green Upload a theme button.

This will prompt an upload box to open. Click inside the upload box to select a theme zip, or drag-and-drop the theme zip into the upload box.

Click Activate to activate the theme.

Services

Custom contact page with Formspree

Formspree is a form backend, API and email service for HTML forms. It is the simplest way to embed custom contact us forms, order forms, or email capture forms on your site.

Add your email address or Formspree ID

You’ll need to locate and open the theme’s ../partials/services/formspree.hbs file in your HTML editor. You need to change XXXXX - which should be replaced with your email address or Formspree ID.

Save your formspree.hbs file in ../partials/services/ folder.

Add custom contact page

Select Pages from the navigation menu on the left of your admin area.

Go to Page Settings Menu within your dashboard.

Then scroll down and at the bottom you will see an option which says Template.

Click on the dropdown menu and select Contact.

Finally, click Publish!

Activate contact form with Formspree
  1. Send a test message from your contact form.
  2. Open your email client and open the received email from Formspree.
  3. Click ‘Activate Form’ to begin receiving submissions from your contact form.

Disqus comments

After you’ve added your site to Disqus, you will be asked to choose a platform.

Select Universal Code from the list of options.

Now that you have your Universal Code copy only your Shortname (example: your-site.disqus.com - copy only your-site). You’ll need to locate and open the theme’s ../partials/services/disqus.hbs file in your HTML editor.

Paste your Shortname - the only thing you need to change is XXXXX - which should be replaced with your own Disqus site-id) and remove brackets {{!-- --}} to use Disqus comments.

Save your disqus.hbs file in ../partials/services folder.

Members

Members Activation

Unzip the theme archive on your computer and locate the file called routes.yaml

Inside Ghost admin, go to the Labs settings area and scroll down until you see the Routes section.

Upload the routes.yaml from this theme.

In Labs settings scroll up until you see the Members section - please enable.

Enable the Allow free member signup option, as well.

You can now optionally create a custom membership page or custom newsletter page.

Login panel with navigation

By default - when Members function is active, login panel it’s enabled.

You could always replace the Sign up field with Membership if you want the user to first get to that page.

For this purpose open ../partials/members_login_panel.hbs file in your HTML editor and find:

              <a class="members-signup" href="{{@site.url}}/signup/">{{t "Sign up"}}</a>

Then replace with:

              <a class="members-signup" href="{{@site.url}}/membership/">Membership</a>

Now create the membership page with /membership/ slug.

Information

The subscription form will not work if the Members feature is not active or if the Allow free member signup option is turned off.

It’s worth noting that currently in Ghost there’s no difference between subscription form members and the signin/signup ones. The subscription form, signin and signup pages are all based on the same form - the only difference is the type of emails received by members:

  • signin form – The default behaviour of members forms. If a member uses this form they will be sent a sign in email, if they are not already a member they will be sent a sign up email instead
  • signup form – If a member uses this form they will be sent a sign up email, if they are already a member they will be sent a sign in email instead
  • subscribe form – If a member uses this form they will be sent a subscribe email, if they are already a member they will be sent a sign in email instead

If you turn the Allow free member signup option off, the subscription forms will no longer work (they will work only when sending the magic login link to the users already in your Members base).

Princing tables

The theme contains three pricing tables: Free, Monthly and Yearly

Free

Members only content is only available to view by members who have signed up and logged in using a magic link sent to their email address.

Monthly and Yearly

This access level only allows members of your site who have an active subscription in Stripe to view the content. This means members will not only need to be logged in, but they also need to have selected a plan that you determined, and entered credit card details in Stripe. Paid-members will automatically be moved back to non-paying members if their subscription is canceled or lapses.

Add feature lists to plans

To add your function list to plan (for example free plan):

  1. You’ll need to locate and open the theme’s ../partials/members_plan_free.hbs file in your HTML editor.
  2. Replace the current <li> ... </li> positions with your own.
  3. Save your members_plan_free.hbs file in ../partials folder.

Proceed in the same way with members_plan_monthly.hbs and members_plan_yearly.hbs.

Remove plan from membership or account page

Example for removing the free plan:

  1. Open custom-membership.hbs or file in your HTML editor.
  2. Remove text {{>members_plan_free}}.
  3. Save your custom-membership.hbs file.

Example for removing the yearly plan:

  1. Open custom-membership.hbs or custom-account.hbs file in your HTML editor.
  2. Remove text {{>members_plan_yearly}}.
  3. Save your custom-membership.hbs or custom-account.hbs file.

Theme translation

How to add any language

Open ../locales folder. Copy and paste en.json, change your lanquage code (e.g.: es for Spanish, fr for French, ro for Romanian). The language_code must be replaced with a valid country code.

Translate the included sentences

Edit the renamed translation file with any plain text editor.

As you can see, it’s usually just plain English on the left, and the language you choose on the right.

Translate the included sentences

Go to the admin panel and select General from the navigation menu on the left of your admin area.

In the Publication info section you'll be able to find Publication Language.

Expand the menu.

Enter your language code.

Click Save Settings to activate.

Reactivate the translation file

If you edited the active language, you have to restart Ghost to make the changes effective.

Alternatively you can go ahead and re-do the previous step.

Don't forget to click on Save Settings before you exit the page so everyhing would work perfectly.
